# CSSの単位について

## px : 固定サイズ

基本的でよく使うが、画面サイズやユーザー設定に柔軟に追従させたい場所では、remや%などの相対単位を優先することが多い

## rem : ルート文字サイズ基準

html要素のfont-sizeを基準にした相対単位。

1remはhtmlのfont-sizeと同じサイズで、1.2remならその1.2倍、0.5remならその半分になる。

font-sizeだけでなく、padding、margin、gap、width、border-radiusなど、長さを指定できるプロパティで使える。

主要なサイズ指定をremにしておくと、htmlのfont-sizeを基準に全体のサイズ感を管理できるため、後から調整しやすく、依存関係も見えやすい。

親要素のfont-sizeには影響されない。親要素のfont-sizeを基準にしたい場合はemを使う。

基本的には、font-size、padding、margin、gapなどはremを使いやすい。一方で、borderの1pxや細かい見た目の微調整はpxのほうが自然なことも多い。

## em : 親や自分の文字サイズ基準

remの自分のfont-sizeを基準にするバージョン

## % : 親要素などのサイズ基準

%はだいたい親基準に対して何％か？でサイズを指定する。
ただし、どの親の何を基準にするかはプロパティごとに違う

例

- font-size:10%;親要素のフォントサイズの10%
- padding:10%;親要素の幅の10%

みたいに幅を見たりフォントサイズを見たりする

## vw : 画面幅基準

表示領域基準
見えている横幅を基準にする単位
1920\*1080　で見てるなら
100vwは1920px
1vwは19.2px

## vh : 画面高さ基準

表示領域基準
見えている縦幅を基準にする単位
1920\*1080　で見てるなら
100vhは1080px
1vhは10.8px

## auto : ブラウザに自動計算させる

autoは単位ではなく、ブラウザに決まったルールで値を自動計算させるキーワード。
autoは「なんとなく任せる」ために使うのではなく、この状況ならこの計算をするとわかっている時に、ブラウザに計算させたい時に使うイメージ

よく使う例

```css
.container {
  width: 800px;
  margin: 0 auto;
}
```

左右のmarginをautoにすると、親要素の幅から自分の幅を引いた余りを左右に分ける。
中央寄せでよく使う。

```css
img {
  width: 100%;
  height: auto;
}
```

画像の幅に合わせて、高さを自動計算する。
画像の縦横比を保ちたい時によく使う。

```css
.box {
  overflow: auto;
}
```

中身がはみ出した時だけスクロールバーを出す。
